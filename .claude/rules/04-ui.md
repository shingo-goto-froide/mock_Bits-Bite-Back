# UI実装ルール

> コードでUIを動的生成する際のルール。
> 過去のプレイテストで繰り返し発生した調整パターンをまとめたもの。

## UI調整の進め方

- **UI確認は2段階で行う**
  - **設計フェーズ**: 画面構成・画面遷移・ダイアログフローを提案 → 承諾
    - 各画面に何があるか（タブ構成、ボタンの種類等）
    - 画面間の遷移（どのボタンでどこへ飛ぶか）
    - ダイアログの有無と流れ（確認→実行→完了 等）
  - **実装フェーズ**: テキストワイヤーフレームで具体的なサイズ(px)・フォント・色・配置を提示 → 承諾 → コード実装
  - これにより「ボタンが大きすぎ」「配置が違う」等の手戻りを防ぐ
- **UIの不具合報告にはスクショをもらう** — どのシーン・どのファイルの問題か特定しやすい
- 同じUIが複数ファイルに重複している場合がある（例: 報酬ダイアログがBattleUIとPrepareUIの両方にあった）。修正前にGrepで全箇所を確認する
- 修正が反映されないと言われたら、別ファイルの同名メソッドを疑う

## コード管理

### スクリプトの行数制限
- **目標: 500行以内、上限: 1000行**
- 1000行を超えそうな場合は分割を提案してから実装を進める
- 分割の単位: タブごと、ダイアログごと、機能ごと（例: PrepareUI → FormationTab, CraftTab, DisassembleTab, SoulTab）

### 共通UIは1箇所にまとめる
- 同じダイアログが複数ファイルに存在しないようにする
- 共通のダイアログ（報酬選択等）は共有ユーティリティクラスに切り出す
- 新規UI作成時にGrepで既存の類似UIがないか確認する

### 本実装ではPrefabを使う
- モックでは動的生成（new GameObject）で進めているが、本実装ではPrefab化する
- コードでのUI生成は最小限にし、レイアウトはEditor上で調整する

## レイアウト

### LayoutGroup内のボタンはサイズ制御が効かない

VerticalLayoutGroup / HorizontalLayoutGroup の `childControlHeight = true` は子の LayoutElement を上書きする。
**固定サイズのボタンが必要な場合は、LayoutGroupの子にせずアンカー配置にする。**

```csharp
// NG: VLG内にボタンを置く → サイズが意図通りにならない
var btn = CreateSimpleButton(vlgParent.transform, ...);

// OK: アンカーで直接配置
var go = new GameObject("Btn");
go.transform.SetParent(parent, false); // VLG外の親
var rt = go.AddComponent<RectTransform>();
rt.anchorMin = new Vector2(0.5f, 0);
rt.anchorMax = new Vector2(0.5f, 0);
rt.pivot = new Vector2(0.5f, 0);
rt.anchoredPosition = new Vector2(0, 12);
rt.sizeDelta = new Vector2(250, 55);
```

### グリッドは中央寄せを忘れない

GridLayoutGroup を使うときは必ず `childAlignment` を設定する。デフォルトは UpperLeft で左寄りになる。

```csharp
glg.childAlignment = TextAnchor.UpperCenter;
```

## サイズの目安

### ボタン

| 用途 | サイズ | フォント |
|---|---|---|
| ダイアログの主要ボタン（装備する/キャンセル等） | 250×55 | 28pt Bold |
| カード内の補助ボタン（魂装備/魂を外す等） | 120×28 | 16pt Bold |
| デバッグ用ボタン | デフォルト | 16pt Bold |

- ボタンは最初から大きめに作る。小さすぎて見づらい指摘が多い
- テキストサイズはボタンの高さに合わせる（高さの約半分が目安）

### ダイアログ

| 要素 | フォント目安 |
|---|---|
| タイトル | 30-32pt Bold |
| サブタイトル・名前 | 24pt Bold |
| 本文・ステータス | 20pt Normal |
| 補足テキスト | 16-18pt |

---

## フォント

- フォントは **NotoSansJP-Regular SDF** を使用
- 以下の記号はフォントに含まれないため使用禁止:
  - `▶` (U+25B6)、`◀` (U+25C0)、`▷` (U+25B7)
- 代替: `>>`, `<<`, `>`, `--` 等のASCII文字を使う

## ダイアログのボタン配置パターン

ダイアログのボタンはVLG外（overlay直下）にアンカーで配置する。
パネルの padding-bottom を広げてボタン用スペースを確保する。

```csharp
// パネルのVLG padding-bottomを広げる
vlg.padding = new RectOffset(25, 25, 20, 55); // bottom=55でボタン分確保

// overlay直下にアンカー配置
// パネル下端のanchor値を基準にする
float btnW = 250f, btnH = 55f, gap = 30f;
var btnGo = new GameObject("Btn");
btnGo.transform.SetParent(overlay.transform, false);
var rt = btnGo.AddComponent<RectTransform>();
rt.anchorMin = new Vector2(0.5f, panelBottomAnchor);
rt.anchorMax = new Vector2(0.5f, panelBottomAnchor);
rt.pivot = new Vector2(1, 0); // 左ボタンはpivot(1,0)、右ボタンはpivot(0,0)
rt.anchoredPosition = new Vector2(-gap * 0.5f, 30);
rt.sizeDelta = new Vector2(btnW, btnH);
```
