## UI設計

> シーン構築の前にレイアウト方針を確定させること。
> 仕様書の「画面操作フロー」が **何をするか**、ここでは **どう見せるか** を定義する。

### 画面レイアウト方針

| シーン | レイアウト | 備考 |
|---|---|---|
| TitleScene | シンプル中央配置 | タイトル + 開始ボタン |
| GameScene | （例：タブ切替 / カラム分割 / グリッド） | 各タブの内容を記述 |

### カード・リストの構成

共通のカード構造を定義する（画像・テキストの配置）。
全画面で統一することでUIの一貫性を保つ。

```
例：
┌────────┬──────────────────┐
│ [画像]  │ 名前（太字）      │
│ 固定    │ パラメータ        │
│ サイズ  │ 能力説明          │
│        │ 素材 etc.         │
└────────┴──────────────────┘
```

### 共通コンポーネント

| コンポーネント | 内容 |
|---|---|
| ヘッダー | Wave表示・所持素材など |
| フッター | アクションボタン・メッセージ |
| タブバー | 画面切替ボタン（使用する場合） |
