## 難易度曲線

v1.0 の7Wave連戦構造から、ダンジョン探索＋エンカウント＋ボス戦に変更。
階層システムにより、ボス撃破→次の階層へと進行する。

### 階層による難易度スケーリング

| 項目 | スケーリング |
|---|---|
| 部屋数 | min: 3+floor, max: 5+floor（上限8〜10） |
| 敵シンボル数 | 2+floor（上限12） |
| 宝箱の素材量 | min: 1+floor-1, max: 2+floor-1 |
| イベント回復量 | 階層で増加（素材量に反映） |
| バトル勝利時の素材報酬 | 2 + floor/2 個 |

### 敵の難易度テーブル（Wave 12段階）

3階層ごとに体感が変わるペースで、全12種のWaveを使用。

| 階層帯 | テーマ | Weak | Medium | Strong | Boss |
|---|---|---|---|---|---|
| 1-4 | スケルトン級 | W1(スケ×2) | W1,W8(スケ×3) | W8,W9(シャドウ+スケ) | W1→W8→W9 |
| 5-7 | 毒・射程登場 | W8,W9 | W9,W2(ゾンビ+スケ),W10 | W2,W10(スケ+弓) | W2→W10→W11 |
| 8-10 | 回復・タンク | W10,W11 | W11,W12(弓×2+僧侶) | W12,W3(ガーディアン+弓×2) | W12→W3→W4 |
| 11+ | フル編成 | W12,W3 | W3,W4 | W4〜W7 | W5〜W7 |

> AutoPopulateWaves() が階層に応じてDungeonConfigSOのWave配列を動的に設定する。

### ダンジョン内の敵配置ルール

- 入口からの距離比率でランクを決定: <35%=Weak, 35-70%=Medium, >70%=Strong
- 宝箱は各部屋に最大1つ配置（同じ部屋に複数の宝箱は出ない）
- イベント部屋は専有（イベントがある部屋にはイベント以外のエンティティを配置しない）
- 入口・ボス部屋から距離3未満のタイルには敵を配置しない

## フィードバックループ

### 短いループ（数秒：1アクション）
- **バトル中:** 攻撃 → ダメージ表示 → HPバー減少（v1.0 と同じ）
- **ダンジョン移動:** 1マス移動 → フォグが晴れる → 新しいマスの内容が見える

### 中くらいのループ（1エンカウント）
- 敵シンボル発見 → 接触するか回避するかの判断 → バトル → 勝利 → HPが減った状態で探索続行
- 宝箱発見 → 素材入手 → 「次のプレイで新しい魔物を練成できる」期待感
- 回復イベント発見 → 回復するかスルーするか（ボス前まで温存?）

### 長いループ（1プレイ）
- 準備（練成・編成）→ ダンジョン探索（複数エンカウント）→ ボス戦 → クリアor全滅
- 「あのルートを通ればボス前にHPが残せたかも」→ 探索戦略の学習
- 「素材が足りない、次は宝箱を優先しよう」→ リソース管理の成長

## バランス数値の設計方針

### ダメージ計算（v1.0 から変更なし）
```
基本ダメージ = 攻撃者の攻撃力
魔法バリアあり（魔法攻撃時）: × 0.5
ガーディアン（魔法攻撃被弾時）: × 1.5
最終ダメージ = 基本ダメージ（小数点以下切り捨て、最低1）
```

### ダンジョン関連パラメータ

| パラメータ | 値 | 根拠 |
|---|---|---|
| マップサイズ | 25×25 グリッド（固定） | Awakeで強制設定。探索に適切な広さ |
| 部屋サイズ | 3〜6 | minRoomSize=3, maxRoomSize=6 |
| 部屋数 | 3+floor 〜 5+floor（上限8〜10） | 階層に応じて増加 |
| 敵シンボル数 | 2+floor（上限12） | 階層に応じて増加 |
| 宝箱数 | 3 | 各部屋に最大1つ |
| イベント数 | 2 | 回復イベントのみ。部屋を専有 |
| 回復量 | 味方全体HP30%回復 | バトル1回分の消耗を半分取り戻せる程度 |
| ボス強化倍率 | HP×2, ATK×1.5 | DungeonConfigSOで設定（ただし現在はWave編成で直接制御） |
| 敵シンボル移動確率 | 50% | ターンごとに50%で1マスランダム移動。プレイヤーに到達でエンカウント |
| バトル勝利時の素材報酬 | 2 + floor/2 個 | 階層に応じて報酬増加 |
| 宝箱の素材量 | 1+floor-1 〜 2+floor-1 個 | 階層で増加 |
| 探索半径 | 3 | プレイヤー周囲3マスが探索済みになる |

### 調整の方向性（v1.0 に追加）
- **敵シンボル数を増やす** → バトル回数が増え、HPリソース管理がシビアになる
- **宝箱数を増やす** → 素材が潤沢になり、練成の選択肢が広がる。難易度は下がる
- **回復量を上げる** → ダンジョンが楽になる。ボス戦に万全で臨める
- **マップサイズを大きくする** → 探索時間が延びる。敵配置が分散するので回避しやすくなる

### DungeonGenerator（pure C# class）

マップのランダム生成ロジック。

| メソッド | 説明 |
|---|---|
| Generate(DungeonConfigSO config) → DungeonMap | マップを生成して返す |
| GenerateRooms(...) → List\<Room\> | 部屋をランダム配置（最低2部屋保証） |
| ConnectRooms(DungeonMap map, List\<Room\> rooms) | L字型通路で部屋同士を接続 |
| PlaceEntrance(DungeonMap map, List\<Room\> rooms) | 入口を最初の部屋に配置 |
| PlaceBossRoom(DungeonMap map, List\<Room\> rooms) | ボス部屋を入口から最も遠い部屋に配置 |
| PlaceEntities(DungeonMap map, DungeonConfigSO config) | 敵・宝箱・イベントを配置（配置ルール付き） |

PlaceEntitiesの配置ルール:
1. イベント部屋を先に専有確保（その部屋のタイルを全て占有済みにする）
2. 宝箱は各部屋に最大1つ（入口・ボス・イベント部屋を除く）
3. 敵は全歩行可能タイルに配置（入口・ボスから距離3以上、距離比率でランク決定）

| 内部クラス | 説明 |
|---|---|
| Room { int x, y, width, height, Vector2Int center } | 部屋データ。Overlaps()で重複判定 |
