## 難易度曲線

v1.0 の7Wave連戦構造から、ダンジョン探索＋エンカウント＋ボス戦に変更。

### ダンジョン内の難易度設計

| 段階 | 内容 | 目的 |
|---|---|---|
| 序盤（入口付近） | 弱い敵シンボル1〜2体 + 宝箱 | 移動操作とエンカウントの流れを理解する |
| 中盤（中間エリア） | 中程度の敵3〜4体 + イベント + 宝箱 | 戦闘でHPが削れ、回復イベントの判断が生まれる |
| 終盤（ボス部屋手前） | 強めの敵1〜2体 | ボス戦前にリソース管理の緊張感 |
| ボス | オーク（強化版）+ 取り巻き | 準備段階の編成が試される最終チェック |

### ダンジョン内の敵シンボル編成

敵シンボルごとにバトル時の敵編成が決まる。ダンジョンに配置される敵シンボルは弱→強の勾配を持つ。

| 敵シンボル種別 | 配置エリア | バトル時の敵編成 |
|---|---|---|
| 弱（スケルトン型） | 入口付近 | スケルトン×2 |
| 弱（ゾンビ型） | 入口〜中間 | ゾンビ×1, スケルトン×1 |
| 中（アーチャー型） | 中間エリア | ガーディアン×1, アーチャー×2 |
| 中（僧侶型） | 中間エリア | 骸骨僧侶×1, ガーディアン×1, ゾンビ×1 |
| 強（結界型） | ボス手前 | 結界師×1, ガーディアン×1, アーチャー×1, スケルトン×1 |
| ボス | ボス部屋 | オーク×1(2枠, 強化), 骸骨僧侶×1, 結界師×1 |

> v1.0 の Wave 1〜7 の編成を流用。ダンジョン内でどの敵にいつ遭遇するかはプレイヤーの探索ルート次第。

## フィードバックループ

### 短いループ（数秒：1アクション）
- **バトル中:** 攻撃 → ダメージ表示 → HPバー減少（v1.0 と同じ）
- **ダンジョン移動:** 1マス移動 → フォグが晴れる → 新しいマスの内容が見える

### 中くらいのループ（1エンカウント）
- 敵シンボル発見 → 接触するか回避するかの判断 → バトル → 勝利 → HPが減った状態で探索続行
- 宝箱発見 → 素材入手 → 「次のプレイで新しい魔物を練成できる」期待感
- 回復イベント発見 → 回復するかスルーするか（ボス前まで温存?）

### 長いループ（1プレイ）
- 準備（練成・編成）→ ダンジョン探索（複数エンカウント）→ ボス戦 → クリアor全滅
- 「あのルートを通ればボス前にHPが残せたかも」→ 探索戦略の学習
- 「素材が足りない、次は宝箱を優先しよう」→ リソース管理の成長

## バランス数値の設計方針

### ダメージ計算（v1.0 から変更なし）
```
基本ダメージ = 攻撃者の攻撃力
魔法バリアあり（魔法攻撃時）: × 0.5
ガーディアン（魔法攻撃被弾時）: × 1.5
最終ダメージ = 基本ダメージ（小数点以下切り捨て、最低1）
```

### ダンジョン関連パラメータ

| パラメータ | 初期値 | 根拠 |
|---|---|---|
| マップサイズ | 30×30 グリッド | 探索に5〜10分。小さすぎると探索感がない、大きすぎると冗長 |
| 部屋数 | 5〜8 | ランダム生成で変動。部屋を通路でつなぐ |
| 敵シンボル数 | 5〜6 | 全部倒す必要なし。回避ルートも可能 |
| 宝箱数 | 2〜3 | 素材1〜2個ずつ。多すぎると練成ループが破綻 |
| イベント数 | 1〜2 | 回復イベントのみ。多すぎるとバトルの緊張感がなくなる |
| 回復量 | 味方全体HP30%回復 | バトル1回分の消耗を半分取り戻せる程度 |
| ボス強化倍率 | HP×2, ATK×1.5 | 通常のオークより明らかに強い。準備なしでは厳しい |
| 敵シンボル移動確率 | 50% | ターンごとに50%で1マス移動。100%だと回避困難 |

### 調整の方向性（v1.0 に追加）
- **敵シンボル数を増やす** → バトル回数が増え、HPリソース管理がシビアになる
- **宝箱数を増やす** → 素材が潤沢になり、練成の選択肢が広がる。難易度は下がる
- **回復量を上げる** → ダンジョンが楽になる。ボス戦に万全で臨める
- **マップサイズを大きくする** → 探索時間が延びる。敵配置が分散するので回避しやすくなる
